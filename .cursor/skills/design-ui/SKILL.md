# デザイン（UI）スキル

## 着手前

- 既存の一覧・詳細・フォームなど**代表画面を 1 つずつ**眺め、色・余白・角・フォント階層の共通パターンをメモする。
- デザイントークンやテーマファイル（Tailwind 設定、CSS 変数など）があれば先に読む。

## 実装時

- 新規 UI は**既存パターンの複製から始め**、足りないところだけ最小限の差分を足す。
- インタラクティブ要素は **focus 可視**・**disabled 時の見た目**・**loading** をセットで考える。
- 文言・ラベル・空状態メッセージは**短く具体的**に。装飾で誤魔化さない。

## 仕上げ

- 主要幅（モバイル／タブレット／デスクトップ）でスクショを想像し、**はみ出し・折り返し崩れ**を疑う。
- 変更したら `docs/dev-diary.md` に「見た目に関わる決定」があれば 1 行残す。
