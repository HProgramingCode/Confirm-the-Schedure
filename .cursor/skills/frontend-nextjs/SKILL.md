# フロントエンド（Next.js）スキル

## 分割の考え方

- まず **Server Component でデータを取り、表示まで**を組む。インタラクションが必要な塊だけ Client に切り出す。
- 同じファイルに `"use client"` とサーバー専用 import を混ぜない（必要なら子コンポーネントに分ける）。

## Zustand

- ストアに入れるのは **UI やクライアント専用の一時状態**が中心。サーバーと同期が必要なら取得タイミングと初期化を明示する。
- 既存 store がある場合は**新規ファイルを増やす前に**拡張可否を検討する。

## データとエラー

- サーバー側で取ったデータの **null / 空配列** を想定し、ユーザー向けの空状態を出す。
- エラーは可能なら **境界（error.tsx）やトースト**など、プロジェクトの既存パターンに合わせる。

## 確認

- 変更後、該当ページのビルドまたは型チェックが通ることを確認する（プロジェクトにスクリプトがあれば実行）。
